# codex高考复习技术文档

## 1. 项目目标与用户需求概述
面向中国高三学生的高考复习网页，覆盖语文、数学、英语、物理、化学、政治六科，重点提升数理化（数学/物理/化学）基础薄弱学生的短期提分效率。核心功能包括：
- 学科体系化复习与高频考点速记
- 基础薄弱学生“短期提分”路径
- Get笔记检索与知识点汇总
- 拍照上传题目，使用视觉模型进行识题与讲解
- 结合AI问答模型输出清晰、分步骤、可操作的解题指导

当前代码基础（位于 `高考复习/`）：
- Vite + React + Tailwind（CDN）
- 已具备学科切换、拍题上传、搜索笔记、错题本等功能
- 现有AI使用 `@google/genai` 的 `gemini-3-pro-image-preview` 与 `gemini-2.5-flash`

本技术文档用于：
- 归纳需求、明确技术方案
- 给出流程图/原型图
- 识别待讨论问题

## 4. 接口与数据方案

### 4.1 Get笔记 Open API（已提供）

- 当前接入（Make Hook）:
  `https://hook.us2.make.com/628uk9k37rq9v8cffmsw4u2ao7kel6l2`
- 官方 OpenAPI Endpoint:
  `https://open-api.biji.com/getnote/openapi/knowledge/search/recall`
- 认证方式:
  `Authorization: Bearer <API_KEY>`
- 版本标识:
  `X-OAuth-Version: 1`
- 示例：

```bash
curl -i "https://open-api.biji.com/getnote/openapi/knowledge/search/recall" \
  -H "Authorization: Bearer <GET_NOTES_API_TOKEN>" \
  -H "X-OAuth-Version: 1" \
  -H "Content-Type: application/json" \
  --data '{"question":"如何学习三角函数","topic_ids":["K0BlyZmn"],"top_k":3}'
```

- 请求示例 JSON（简化）：

```json
{
  "question": "三角函数的基础概念是什么",
  "topic_id": "K0BlyZmn",
  "top_k": 3
}
```

- 请求示例 JSON（带追问）：

```json
{
  "question": "能举个具体例子吗",
  "topic_ids": ["K0BlyZmn"],
  "history": [
    { "content": "三角函数是什么", "role": "user" },
    { "content": "三角函数描述角度与边长关系", "role": "assistant" }
  ],
  "top_k": 3,
  "intent_rewrite": false,
  "select_matrix": false
}
```

- 请求字段:

| 字段 | 类型 | 说明 |
|---|---|---|
| question | string | 查询问题/关键词 |
| topic_id | string | 知识库ID（优先使用） |
| topic_ids | string[] | 知识库ID列表（当前只支持1个） |
| top_k | int | 筛选相似度最高的 N 个（默认10） |
| intent_rewrite | bool | 问题意图重写（默认false） |
| select_matrix | bool | 对结果进行重选（默认false） |
| history | array | 搜索记录（用于追问） |

- 返回结构（来自官方文档）
  - 顶层结构：
    - `h`: 元信息（示例字段：`c`、`e`、`s`、`t`、`apm`）
    - `c.data`: 召回结果数组
  - `c.data[]` 字段：

| 字段 | 类型 | 说明 | 备注 |
|---|---|---|---|
| id | string | 召回对应资源ID |  |
| title | string | 召回对应资源标题 | 可能为空 |
| content | string | 召回内容 |  |
| score | float | 得分 |  |
| type | string | 召回资源类型 | FILE / NOTE / BLOGGER |
| recall_source | string | 召回来源 | embedding / keyword |

- 响应示例 JSON（节选）：

```json
{
  "h": {
    "c": 0,
    "e": "",
    "s": 1741253826,
    "t": 2099,
    "apm": "4962fec80013253"
  },
  "c": {
    "data": [
      {
        "id": "PAO/hakxI/mel6b+g+SuCA==",
        "title": "一口气了解关税",
        "content": "关税的基本概念与运行机制...",
        "score": 0.09380767438923665,
        "type": "NOTE",
        "recall_source": "embedding"
      }
    ]
  }
}
```

- 限制说明（公测期）：
  - QPS: 2
  - 日调用上限: 5000 次
  - 公测期间接口能力可能调整

### 4.5 Gemini 计费与限额策略（独立章节）

目标：在可控成本内提供稳定答题服务，并避免高峰期超额请求。

**策略建议：**
- 计费账号统一由后端配置，前端不暴露 Key
- 设定每日/每用户调用上限（防滥用）
- 对失败请求进行快速重试与降级（如简化回答）
- 记录请求量与成本指标（按科目、功能统计）

**限额与节流（建议参数）：**
- 单用户并发: 1-2
- 单用户 QPS: 0.5-1
- 全局 QPS: 视预算与配额调整

**成本优化：**
- 题图识别与文字讲解分离，优先小模型完成结构化识题
- 对重复问题进行缓存命中（同一题图或相似题）
- Get笔记结果优先，AI作为补充与结构化总结

## 8. 技术方案优化建议

### 8.1 架构优化

| 方向 | 建议 | 价值 |
|---|---|---|
| 前端 | 抽象模型服务层 | 便于替换 nano banana pro2 / 其他问答模型 |
| 请求 | 增加请求状态/错误提示统一组件 | 降低用户迷惑 |
| 存储 | 错题本支持云同步（可选） | 多设备复习连续性 |
| 模型 | 拆分“识题”和“讲解”提示词 | 解析更稳定，逻辑更清晰 |

### 8.2 内容优化

| 方向 | 建议 | 价值 |
|---|---|---|
| 提分策略 | 加入“常错点提醒”模块 | 针对基础薄弱 |
| 学科策略 | 数理化提供“公式卡片” | 记忆效率提升 |
| 结果显示 | 输出“错因诊断 + 纠正方法” | 强化训练闭环 |

### 8.3 Get笔记 API 迁移方案（Make Hook -> 官方 OpenAPI）

目标：在不影响现有功能的前提下，平滑切换到官方 OpenAPI，并保留回退能力。

| 阶段 | 动作 | 关键点 | 回退策略 |
|---|---|---|---|
| 阶段1 评估 | 保留 Make Hook 现状，新增官方 OpenAPI 配置 | 新增环境变量与开关；记录响应结构 | 出错即回退到 Make Hook |
| 阶段2 双轨 | 请求并行（灰度比例） | 对比响应字段与内容质量 | 失败自动回退 |
| 阶段3 切换 | 默认走官方 OpenAPI | 监控 QPS/限额，做好节流 | 保留 Make Hook 作为备用 |
| 阶段4 清理 | 视情况移除 Make Hook | 完成监控与告警 | 无 |

迁移配置建议：
- `GET_NOTES_PROVIDER=make|openapi`
- `GET_NOTES_API_KEY` 放后端环境变量
- 统一结果归一化（字段映射到 `answer` 或 `content` 展示）
